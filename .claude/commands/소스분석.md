# /소스분석 - Phase 1: 소스별 분석 및 요약 생성

분석할 소스를 선택하세요: $ARGUMENTS
(예: confluence, jira, figma, code, all)

## 실행 규칙

1. CLAUDE.md의 "소스 분석 규칙"을 반드시 따를 것
2. 분석 결과는 `_summaries/` 폴더에 저장
3. 요약은 A4 1-2페이지 분량으로 간결하게
4. 테스트 관점에서 중요한 정보 위주로 추출

## 소스별 분석 방법

### confluence
- Atlassian MCP(`mcp__atlassian__confluence_get_page`)를 사용하여 페이지 분석
- 펼치기 UI 내 콘텐츠, 순서도 분기, 연결된 하위 페이지 모두 분석
- 젠데스크 웹 관련 내용은 제외
- 결과 저장: `_summaries/confluence_summary.md`

### jira
- Atlassian MCP(`mcp__atlassian__jira_get_issue`)를 사용하여 이슈 분석
- Epic과 하위 Task 모두 분석
- 연결된 GitHub PR도 확인
- 결과 저장: `_summaries/jira_summary.md`

### figma
- Figma MCP(`mcp__figma__get_design_context`)를 사용하여 디자인 분석
- 이미지/색상 무시, 입력/버튼/검증/화면전환만 추출
- 플로우차트 텍스트 → 분기 커버리지 고려
- AS-IS/TO-BE 차이점 비교
- 결과 저장: `_summaries/figma_summary.md`

### code
- GitHub MCP(`mcp__github__get_pull_request_files`)를 사용하여 PR 분석
- 로컬 코드 저장소(`/Users/Claude/wadiz_RC`)도 함께 분석
- 결과 저장: `_summaries/code_summary.md`

### all
- 위 4가지를 서브에이전트(Task tool)로 **병렬 실행**
- 각 소스를 별도 서브에이전트에게 할당하여 동시 분석
- 모든 분석 완료 후 `_summaries/` 폴더에 4개 요약 파일 생성
