# /TC리뷰 - 테스트케이스 리뷰 및 커버리지 개선

리뷰 대상: $ARGUMENTS
(예: 서포터WAi_테스트케이스_v2.xlsx, 또는 latest)

## 리뷰 관점 (5단계)

### 1단계: 커버리지 갭 분석
- `_summaries/` 요약 파일과 기존 TC를 대조
- 요약에는 있으나 TC에 빠진 기능/시나리오 식별
- 시트별 커버리지 비율 산출

### 2단계: ISTQB 기법 적용 검증
- 동치 분할: 입력 데이터의 유효/무효 클래스가 모두 커버되었는지
- 경계값 분석: 경계 조건(파일 크기 50MB, 파일 수 10개 등)이 테스트되는지
- 상태 전이: 문의 상태(문의완료→답변완료→문의해결) 전이가 모두 커버되는지
- 조건 기반: 복합 조건(로그인+서포터+주문이력 있음 등) 조합이 충분한지

### 3단계: 플랫폼 교차 검증
- iOS / Android / Web 각 플랫폼별 TC 존재 여부 확인
- 플랫폼 특화 동작(웹뷰 인터페이스, 스와이프 등) 커버 여부

### 4단계: 예외/엣지케이스 점검
- 네트워크 오류, 타임아웃
- 동시 접속, 세션 만료
- 비정상 입력(특수문자, 초장문, 빈값)
- 다국어(ko/en/ja/zh) 전환 시나리오

### 5단계: 리뷰 리포트 생성
출력 형식:
```
## TC 리뷰 결과

### 요약
- 현재 TC 수: X개
- 커버리지 갭: Y건
- 추가 권장 TC: Z건

### 시트별 분석
| 시트 | 현재 TC | 누락 영역 | 추가 권장 |
|------|---------|-----------|-----------|

### 구체적 추가 권장 TC 목록
1. [시트] 테스트제목 - 사유
2. ...

### 개선 우선순위 (높음/중간/낮음)
```

## 자동 보완 옵션
리뷰 후 "보완 실행"을 요청하면 `/TC생성` 커맨드를 활용하여 누락된 TC를 기존 xlsx에 추가합니다.
